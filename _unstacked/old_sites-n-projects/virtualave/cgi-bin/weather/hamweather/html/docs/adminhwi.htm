Administration : Include Files

HAMweather allows you to easily insert files into a template during output. This can be used to insert a common header or footer into your weather output. Thus, allowing you to simply change a couple files to personalize your weather site. (Those familiar with SSI will know the power of this option.)

The process of using HAMweather include (HWI) is a two step process.

Step 1 Predefine Include Files

You must first predefine what files can be included and give each one a 'nickname'. The reason for this is better security. You can do this by using the web based administration HAMweather Include page. HAMweather comes with the following include files preset:

HAMweather Include Presets
Nickname Filename
header header.html
footer footer.html
nav1 nav1.html
nav2 nav2.html
nav3 nav3.html
form1 form1.html

The include files are set to be in the /templates/include directory of your HAMweather install.

To administrate the HAMweather Include presets, click the 'Include Files' link in the administration options at the top left of all administration pages.

The Include Files Path

The Include Files Path is the location where the files to be included are located.   The default is the /templates/include directory if your HAMweather installation.   If you want to change this path, modify the path in the text box labeled 'Inlcude Files Path'. Also, check the checkbox that is to the left of the 'Include Files Path'. Next click the submit button and then the path will be changed.

The Include Files

To add an include file, type in the include file nickname in a blank left most text box.  Then type in the filename of the actual file ot be included in the rightmost text box.  Finally check the checkbox to the left of the include file nickname you added.  When you submit the form the include file will be added to the users include file list.   To remove an include file just unclick the checkbox to the left of the un wanted include file  and submit the form.
