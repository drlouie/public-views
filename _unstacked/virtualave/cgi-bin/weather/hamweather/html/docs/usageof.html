Controlling the HAMweather Output

HAMweather allows you to control where the output goes.  There are three modes, the default and most commonly used is to send the output to the web browser, but you can have HAMweather output to a file or even both, the web and a file.

The output of HAMweather is controlled using the 'om' (Output Method) and the 'of' (Output Filename) parameters. 

The 'om' parameter sets what method of output HAMweather should use.  If not set or if set to zero (0) then the output will be sent to the web.  If set to one (1)   then output will be sent to both the web and a file.  Finally if set to two (2), the output will be sent to a file only.

When you want the output to goto a file, then you should also set the 'of' parameter.   this shoudl be set to the nickname of a file to output to.  You can used the web based administration to set the various files that can be outputted.  Also, if you do not specify the 'of' parameter, then HAMwetaher will use the default ouptut file name.  This filename can also be set using the web based administration.

Examples of Use:

To send output to the web simply do not specify the 'om' parameter in the URL:
http://www.yourdomain.com/path/to/hamweather.cgi?place=roanoke&state=va&forecast=zandh

To send output to the default file only then include the 'om' parameter in the URL:
http://www.yourdomain.com/path/to/hamweather.cgi?place=roanoke&state=va&forecast=zandh&om=2

To send output to the test output file then include the 'om' and 'of' parameters in the URL:
http://www.yourdomain.com/path/to/hamweather.cgi?place=roanoke&state=va&forecast=zandh&om=2&of=test

To send output to both web and test output file use the following url:
http://www.yourdomain.com/path/to/hamweather.cgi?place=roanoke&state=va&forecast=zandh&om=1&of=test
